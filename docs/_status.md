# Docs Status

The repository is using the indexed docs model under `docs/`.

Current reality:

- `_index.md` is the navigation layer.
- `README.md` is the human mental-model entrypoint for each docs folder.
- `_status.md` and `_decisions.md` are part of the folder-level docs shape when
  their information is needed.
- `docs/process/` exists for the repository-wide documentation workflow.
- `docs/DICTIONARY.md` exists as the canonical terminology dictionary for the
  docs tree.

Known gaps:

- The docs system is in place, but the repository does not yet use additional
  top-level docs areas beyond the workflow and dictionary files.
