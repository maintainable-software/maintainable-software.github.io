# Process Overview

This area explains how documentation is structured, discovered, maintained, and
kept consistent in this repository.

It exists so both humans and agents can find the smallest useful set of docs,
understand how those docs are supposed to be used, and keep the docs tree
synchronized as repository truth changes.

Read the focused docs here when you need details about routing, the indexed
docs system, [conceptual simplicity](../DICTIONARY.md#conceptual-simplicity),
anti-[complecting](../DICTIONARY.md#complecting) rules, transitional states,
[co-located docs](../DICTIONARY.md#co-located-docs), or the dictionary workflow
for terminology used in the docs tree.
