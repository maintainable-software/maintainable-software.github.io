# Docs Overview

This `docs/` tree defines the repository's documentation system and the
workflow for keeping it consistent.

Use it to understand the docs model, the agent-facing maintenance rules, and
the conventions that keep [global docs](DICTIONARY.md#global-docs),
[co-located docs](DICTIONARY.md#co-located-docs), and terminology aligned.

Start with [process/README.md](process/README.md) if you want the mental model
for the documentation workflow itself.

Read [process/conceptual-simplicity.md](process/conceptual-simplicity.md) when a
task affects conceptual boundaries, responsibility decomposition, docs
structure, or implementation organization.

Use [DICTIONARY.md](DICTIONARY.md) when repository terminology matters or when
docs prose may introduce, rename, or redefine important terms.
